  Goal

The game is based on 24 fields on a grid organized on X-axes A..G and Y-axes 1..7

Image Source Wikipedia The second board is only to see as example for the fields, the lines are not corresponding the game nine men's morris

The game is played in 3 phases:

1) Placing stone Each player has 9 stones, which are placed turn by turn and one per turn
2) Moving stone After all stones are placed, the player can move one stone per turn and to one adjusting free field
3) Flying When a player has only 3 stones left, he can move his stones to any free field

Capture When a player form by placing or movement a mill, means three of his stones in a vertical or horizontal line, he has to take a piece of the opponent, this in prioritization, an opponent free stone (not in a mill) or when no free stone is available any stone.

Winning
The player who takes all but 2 stones from the opponent or block the opponent that he can no more move wins.

  Rules

Mill

A mill is an alignment of tree stones in vertical or horizontal by placing or moving a stone on a previous empty field of the line. When creating a mill, the player will take a stone from the opponent. First only stones not forming a mill are allowed to be taken. When all stones are bound in a mill, the player can take any stone of the opponent.

Placing stones

PLACE ; field1
PLACE: The command to place a stone
field1: The field on the board to place the stone. It must be free. As Example A1

PLACE&TAKE ; field1 ; field2
PLACE&TAKE: The command to place a stone. This command has to be used when placing forms a mill.
field1: The field on the board to place the stone. It must be free. As Example A1
field2: The field on the board with the opponents stone. As Example A2


Move stones

MOVE ; field1 ; field2
MOVE: The command to move a stone
field1: The field on the board containing the stone the player want to move. As Example A1
field2: The field on the board to move the stone to. It must be free and adjusting the field1. As Example D1

MOVE&TAKE ; field1 ; field2 ; field3
MOVE&TAKE: The command to move a stone. This command has to be used when moving forms a mill.
field1: The field on the board containing the stone the player want to move. As Example A1
field2: The field on the board to move the stone to. It must be free and adjusting the field1. As Example D1

field3: The field on the board with the opponents stone. As Example A2

Flying

MOVE ; field1 ; field2
MOVE: The command to move a stone
field1: The field on the board containing the stone the player want to move. As Example A1
field2: The field on the board to move the stone to. It can be any field on the board but it must be free. As Example D1

MOVE&TAKE ; field1 ; field2 ; field3
MOVE&TAKE: The command to move a stone. This command has to be used when moving forms a mill.
field1: The field on the board containing the stone the player want to move. As Example A1
field2: The field on the board to move the stone to. It can be any field on the board but it must be free. As Example D1

field3: The field on the board with the opponents stone. As Example A2

  Game Input

Initialization Input
playerId Integer value for playerId (0,1)

Input for one game turn
Line 1: opMove String Last move of opponent. In case of no move yet done, it returns -
Line 2: board String Current Board and state(0:Player0 | 1:Player1 | 2:Empty) in format field:state separated by ;
Line 3: nbr int Number of valid commands provided as input.
Line X: command PLACE;A1 X corresponding nbr of previous input commands.

Output for one game turn
Line 1: command PLACE;A1 One single line containing one COMMAND and the corresponding fields separated with ";"

Victory Conditions
  • Taking an opponent stone and your opponent has only 2 stones on the board.
  • You block all movements of your opponent.
Loss Conditions
  • Not responded in time (50ms)
  • Provide a invalid command
  • Provide a invalid move or move (invalid stone selected, invalid target provided)
  • Capture a wrong stone, or not capturing when forming a mill
