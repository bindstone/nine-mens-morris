  Goal

The game is based on a grid with 24 fields organized on X-axes A..G and Y-axes 1..7

Image Source Wikipedia The board on the right is meant to show the system of coordinates. The lines do not correspond to the game nine men's morris.

The game has a Debug Mode, showing the field positions.

The game is played in 3 phases:

  1. Place stone Each player has 9 stones. The stones are placed on a free field once per round.
  2. Move stone When all stones are placed, the player can move one stone per turn to an adjacent free field.
  3. Flying When a player only has 3 stones left, he can move one of his stones to any free field.
Capture
When by placement or movement a player forms a mill, meaning three of his stones in a vertical or horizontal line, he must take an opponent stone. Stones that are in a mill can only be taken when no other stones are available to capture.

Winning
The player who takes all but 2 stones from the opponent or prevents the opponent to do any move next turn, wins.

Draw
When after 200 movements no player has won, the game is declared to be a draw.

  Rules

Mill

A mill is an alignment of three stones in vertical or horizontal by placing or moving a stone on a previously empty field of the line.
When creating a mill, the player takes a stone from the opponent. First, only stones not forming a mill are allowed to be taken.
When all stones are bound in a mill, the player can take any stone of the opponent.

Place stone

PLACE ; field1
PLACE: The command to place a stone.
field1: The field on the board to place the stone. It must be free. As example A1.

PLACE&TAKE ; field1 ; field2
PLACE&TAKE: The command to place a stone. This command has to be used when placing forms a mill.
field1: The field on the board to place the stone. It must be free. As example A1.
field2: The field on the board with the opponent's stone to take. As example A2.

Move stone

MOVE ; field1 ; field2
MOVE: The command to move a stone.
field1: The field on the board containing the stone the player wants to move. As example A1.
field2: The field on the board to move the stone to. It must be a free and adjacent field. As example D1.

MOVE&TAKE ; field1 ; field2 ; field3
MOVE&TAKE: The command to move a stone. This command has to be used when moving forms a mill.
field1: The field on the board containing the stone the player wants to move. As example A1.
field2: The field on the board to move the stone to. It must be free and adjusting to field1. As example D1.
field3: The field on the board with the opponent's stone to take. As example A2.

Flying

MOVE ; field1 ; field2
MOVE: The command to move a stone.
field1: The field on the board containing the stone the player wants to move. As example A1.
field2: The field on the board to move the stone to. It can be any field on the board but it must be free. As example D1.

MOVE&TAKE ; field1 ; field2 ; field3
MOVE&TAKE: The command to move a stone. This command has to be used when moving forms a mill.
field1: The field on the board containing the stone the player wants to move. As example A1.
field2: The field on the board to move the stone to. It can be any free field on the board. As example D1.
field3: The field on the board with the opponent's stone to take. As example A2.

  Game Input

Initialization Input
playerId Integer value for playerId (0,1)

Input for one game turn
Line 1: opMove String Last move of the opponent. In case of no move yet done, it returns -
Line 2: board String Current Board, field and state(0:Player0 | 1:Player1 | 2:Empty) in format field: state separated by ;
Line 3: nbr int Number of valid commands provided as input.
Line X: command PLACE;A1 X corresponding nbr of previous input commands.

Output for one game turn
Line 1: command PLACE;A1 One single line containing one COMMAND and the corresponding fields separated with ";"

Victory Conditions
  • Taking an opponent stone and your opponent has only 2 stones on the board.
  • You block all movements of your opponent.
Loss Conditions
  • Not responding in time (50ms).
  • Provide an invalid command.
  • Provide an invalid move or field (invalid stone selected, invalid target field provided).
  • Capture a wrong stone, or not capturing when forming a mill.
When after 200 moves no one has won, the game is declared a draw.
