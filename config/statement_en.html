This is a Community Contribution game.

For this challenge, two leagues are available. There are no rule or game changes between the leagues.
Nine men's morris is a classical strategic board game, like checkers or chess. More information on Wikipedia.

  Goal

The game is based on a grid with 24 fields organized on X-axes A..G and Y-axes 1..7



In Debug the field positions are shown on the board.

The game is played in 3 phases:
  1. Place stone Each player has 9 stones. On each turn a player places one stone on a vacant field.

  2. Move stone When all 18 stones are placed, the player can move one stone/round to an adjacent vacant field.

  3. Flying Once a player has only three pieces left, he can move a stone to any vacant position on the board and not just to an adjacent vacant field.
Capture
When a player forms a mill with a placement or movement, he must take an opponent stone. A mill means three stones from the player forms a vertical or horizontal line.
The order to take a stone is first only stones not in a mill. When all opponent stones are in a mill, the player can take any stone.

Winning
The player who takes all but 2 stones from the opponent or prevents the opponent to do a move next turn, wins.

Draw
After 200 movements and no player has won, the game is declared as draw.

  Commands

A separator ";" is used between action and fields.

Place stone

PLACE ; field1
PLACE: The command to place a stone.
field1: The field on the board to place the stone. It must be vacant. As example A1.

PLACE&TAKE ; field1 ; field2
PLACE&TAKE: The command to place a stone. This command has to be used when placing forms a mill.
field1: The field on the board to place the stone. It must be vacant. As example A1.
field2: The field on the board with the opponent's stone to take. As example A2.

Move stone

MOVE ; field1 ; field2
MOVE: The command to move a stone.
field1: The field on the board containing the stone the player wants to move. As example A1.
field2: The field on the board to move the stone to. It must be a vacant and adjacent field. As example D1.

MOVE&TAKE ; field1 ; field2 ; field3
MOVE&TAKE: The command to move a stone. This command has to be used when moving forms a mill.
field1: The field on the board containing the stone the player wants to move. As example A1.
field2: The field on the board to move the stone to. It must be vacant and adjusting to field1. As example D1.
field3: The field on the board with the opponent's stone to take. As example A2.

Flying

MOVE ; field1 ; field2
MOVE: The command to move a stone.
field1: The field on the board containing the stone the player wants to move. As example A1.
field2: The field on the board to move the stone to. It can be any field on the board but it must be vacant. As example D1.

MOVE&TAKE ; field1 ; field2 ; field3
MOVE&TAKE: The command to move a stone. This command has to be used when moving forms a mill.
field1: The field on the board containing the stone the player wants to move. As example A1.
field2: The field on the board to move the stone to. It can be any free field on the board. As example D1.
field3: The field on the board with the opponent's stone to take. As example A2.

  Game Input

Initialization Input
playerId Integer value for playerId (0,1)

Input for one game turn
Line 1: opMove String Last move of the opponent. In case of no move yet done, it returns -
Line 2: board String Current Board, field and state(0:Player0 | 1:Player1 | 2:Empty) in format field: state separated by ;
Line 3: nbr int Number of valid commands provided as input.
Line X: command PLACE;A1 X corresponding nbr of previous input commands.

Output for one game turn
Line 1: command PLACE;A1 One single line containing one COMMAND and the corresponding fields separated with ";"

Victory Conditions
  • Take an opponent stone and your opponent has only 2 stones on the board.
  • Block all of opponents stones, to do a valid move.
Loss Conditions
  • Not responding in time (50ms).
  • Provide an invalid command.
  • Provide an invalid move or field (invalid stone selected, invalid target field provided).
  • Capture a wrong stone, or not capturing when forming a mill.

If after 200 moves no one has won, the game is declared a draw.
